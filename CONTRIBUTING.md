# Contributing to resonanceOS-streamer

Thank you for your interest in contributing to our project! Here's how you can help:

## Contribution Guidelines
1. **Fork the repository** to your own account.
2. **Create a new branch** for your feature or bug fix.
3. **Make your changes** and commit them with clear commit messages.
4. **Push your branch** to your forked repository.
5. **Open a pull request** against the main repository.

## Code of Conduct
We expect all contributors to adhere to our code of conduct. Please be respectful and considerate to others at all times.

## Pull Request Process
1. Before submitting your pull request, please ensure that your code passes all tests.
2. Review your changes for accuracy and completeness.
3. Provide a clear description of your changes in the pull request.
4. Pay attention to the feedback from project maintainers and be prepared to make changes as requested.

## Development Setup Instructions
To set up your development environment, follow these steps:
1. Clone the repository: `git clone https://github.com/rozza591/resonanceOS-streamer.git`
2. Navigate to the project directory: `cd resonanceOS-streamer`
3. Install dependencies: `npm install` (or your package manager of choice)
4. Run the development server: `npm start`

We appreciate your interest in contributing to our project! If you have any questions, feel free to open an issue or reach out to us directly.
